# Target Accuracy Evaluator

## What
Create `js/core/evaluators/target-accuracy.js` — extracts and generalizes the scoring logic from the existing `song-engine.js` into a standalone evaluator that the exercise runtime can use.

## Detailed Requirements
- Interface: `create(config)` returns `{ onPitch(pitchData, targetNote), getScore(), getNoteScores(), reset() }`
- Extract scoring logic from song-engine.js: accuracy calculation (sqrt curve), hold percentage, cents deviation tracking
- Support player-driven mode: note advances when player sustains target pitch for `holdMs` (default 300ms), no time limit per note, track time-to-hit as metric
- Support fixed-tempo mode: note advances on timer (backward compat with current scroll behavior)
- Generous scoring preserved: sqrt accuracy curve, 70/30 accuracy/hold split, silence doesn't penalize
- Output measurements: cents-avg, notes-hit-pct, per-note scores array, time-per-note (for player-driven)
- Pure logic module — receives pitch data, returns scores, no DOM or audio dependencies

## Context
This is Phase 1.2 of IMPLEMENTATION_PLAN.md. The evaluator is the bridge between the pitch stream and meaningful exercise feedback. It must preserve the existing generous scoring philosophy while adding player-driven timing support.

## Builder Guidance
- Certainty level: Firm — scoring behavior is validated and the user likes it
- Read `js/audio/song-engine.js` closely — the scoring logic there is the starting point
- Player-driven timing is critical: notes WAIT for the player, speed is a metric not a gate
- Keep the evaluator stateless between exercises (reset() clears everything)

## Plan

1. Create `js/core/evaluators/` directory
2. Create `js/core/evaluators/target-accuracy.js` with:
   - `create(config)` factory that returns evaluator instance
   - Config accepts: tolerance (cents), holdMs (player-driven), scoring weights
   - Per-note accumulators: totalFrames, inTuneFrames, centsSum, centsCount, bestCents, hitTimeMs, holdStartTime
   - `onPitch(pitchData, targetNote)` — processes each pitch frame:
     - Computes exactCents = (pitchMidi - targetMidi) * 100 + pitchCents
     - Tracks in-tune frames (within tolerance)
     - Player-driven: tracks consecutive in-tune time, returns `{ advance: true }` when held for holdMs
     - Fixed-tempo: just accumulates, no advance signal
   - `advanceNote()` — called by runtime to move to next note, finalizes current note score
   - `getScore()` — returns overall score (average of note scores)
   - `getNoteScores()` — returns per-note score array
   - `getMeasurements()` — returns standardized measurement object
   - `reset()` — clears all state
3. Port scoring formula from song-engine.js:
   - rawAccuracy = max(0, 1 - avgCents / (tolerance * 1.5))
   - accuracy = sqrt(rawAccuracy) — generous curve
   - score = round((accuracy * 0.7 + holdPct * 0.3) * 100) — 70/30 split
   - Silence doesn't penalize (only pitch frames count)
4. Player-driven timing logic:
   - Track `holdStartTime` when player is in-tune for consecutive frames
   - When in-tune duration >= holdMs, signal advance
   - Track `timeToHitMs` — time from note start to first in-tune frame
5. Export: named export `createTargetAccuracyEvaluator`
