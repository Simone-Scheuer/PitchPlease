# Renderer Base

## What
Create `js/renderers/renderer-base.js` — shared canvas setup, DPI scaling, resize handling, color/font helpers, and documented renderer interface.

## Detailed Requirements
- Shared canvas utilities as exported functions (not a class to inherit from):
  - `setupCanvas(canvasElement)` → `{ canvas, ctx, width, height, dpr }` with DPI scaling
  - `resizeCanvas(canvas, ctx)` → updated dimensions
  - `centsToColor(absCents)` → CSS color string using tokens.css values (green ≤10, yellow ≤25, red >25)
  - `drawNoteLabel(ctx, note, octave, x, y, opts)` — shared note label rendering
- Document the renderer interface via JSDoc (all renderers must implement):
  - `init(canvasElement, exerciseConfig)` — set up canvas and exercise-specific state
  - `start(config)` — called when exercise begins
  - `update(state)` — called each rAF frame by exercise runtime. State = { pitchData, targetNote, cursor, noteCount, elapsed, evaluatorResult, exerciseState, iteration }
  - `stop()` — halt rendering
  - `destroy()` — full cleanup (remove listeners, release canvas)
  - `onCountdown(secondsLeft)` — show countdown overlay (optional)
  - `onLoopRestart()` — reset for next iteration (optional)
- Color constants matching tokens.css: BG (#0d0d0d), IN_TUNE (#4ecdc4), CLOSE (#ffe66d), OFF (#ff6b6b), TEXT (#f0f0f0), DIM (#666), GRID (#1a1a1a)
- Font constants matching tokens.css font families
- Pure utility module — no state, no event bus

## Builder Guidance
- Certainty level: Firm
- Read css/tokens.css for exact color values
- Read js/components/game-canvas.js for existing DPI scaling and color patterns to extract
- This is a utility module, not a base class — renderers use these functions, don't extend a class
