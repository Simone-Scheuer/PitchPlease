# Exercise Config Schema

## What
Create `js/core/exercise-schema.js` — the declarative exercise config format that all exercises are defined as. Includes type definitions (JSDoc), validation, defaults, and factory helpers.

## Detailed Requirements
- Define ExerciseConfig shape with all fields from PRD: id, type, name, description, context, evaluator, renderer, timing, audio, duration, loop, measures, skills
- Exercise types: 'sustained', 'sequence', 'reactive', 'echo', 'free'
- Timing modes: 'player-driven' (default), 'fixed-tempo', 'auto-tempo', 'indefinite'
- Player-driven timing is the DEFAULT for all exercises per design principles
- Validation function: `validateExercise(config)` returns `{ valid, errors }`
- Default values for optional fields (timing defaults to player-driven, loop defaults to true, etc.)
- Factory helper: `createSequenceExercise(params)` — shorthand for the most common exercise type
- Pure data module — no DOM, no audio, no side effects

## Context
This is Phase 1.1 of IMPLEMENTATION_PLAN.md. The schema is the foundation — exercise runtime, evaluators, renderers, session runner, and session generator all depend on this format.

## Builder Guidance
- Certainty level: Firm — the schema is well-defined in the PRD
- Reference PRD.md "Exercise Config Schema" section for the complete field spec
- Use JSDoc for type definitions (no TypeScript — no build step)
- Keep it minimal — just the schema, validation, and defaults
